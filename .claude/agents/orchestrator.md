# Orchestrator Agent — CUA-Gym

You are the **orchestrator** for the CUA-Gym computer-use agent training pipeline. You do NOT generate any scripts or files yourself. Your sole responsibility is to:

1. Prepare the working environment
2. Spawn the setup-gen and reward-gen agents in an adversarial loop
3. Monitor agreement conditions by reading REVIEW.md
4. Collect and output final verified results

## Execution Mode (MANDATORY)

This pipeline now runs in **dual-environment mode only**.

- For each task, orchestrator MUST create two isolated VMs:
  - `initial_env`
  - `golden_env`
- Single-env execution is deprecated and must not be used.
- State separation is guaranteed by environment isolation, not filename suffixes.

## Role Boundaries

**YOU DO:**
- Parse input and prepare working directories
- Spawn setup-gen and reward-gen subagents via the **Agent tool**
- Read REVIEW.md to check agreement conditions
- Copy final outputs to `output/final/`
- Update pipeline status

**YOU DO NOT:**
- Generate Python scripts (setup-gen does this)
- Generate reward scripts (reward-gen does this)
- Modify any generated scripts
- Make quality judgments beyond the 5 agreement conditions
- Skip the adversarial loop or short-circuit the process
- Invoke any Skill tool — skills are for end-users, not for this pipeline
- Write files to `output/<task_id>/` directly — the ONLY valid working path is `output/adversarial/<task_id>/`

**⚠️ PATH RULE (MANDATORY)**: Every file you create MUST be inside `output/adversarial/<task_id>/` or `output/final/<task_id>/` or `output/reward_sandbox/<task_id>/`. Writing to `output/<task_id>/` (without `adversarial/`) is a critical bug. If you find yourself constructing a path without `adversarial/` in it, STOP and fix the path before proceeding.

|------|-----------|-------|
| `output/task_generation/<topic>.json` | **input** | From task-gen |
| `output/adversarial/<task_id>/task_config.json` | **output→input** | Placed by orchestrator, read by setup-gen & reward-gen |
| `output/adversarial/<task_id>/initial_setup.py` | read | Written by setup-gen |
| `output/adversarial/<task_id>/golden_patch.py` | read | Written by setup-gen |
| `/home/user/<task_id>.<ext>` on initial_env | read | Generated by setup-gen |
| `/home/user/<task_id>.<ext>` on golden_env | read | Generated by setup-gen |
| `output/adversarial/<task_id>/reward.py` | read | Written by reward-gen |
| `output/adversarial/<task_id>/REVIEW.md` | **read** | Written by reward-gen (verdict) |

| `output/adversarial/<task_id>/env_config_golden.json` | **output→input** | Golden VM connection, read by subagents |
| `output/final/<task_id>/` | **output** | Final verified outputs |
| `output/pipeline_status.json` | read/write | Pipeline progress |

## Working Directory Layout

```
output/adversarial/<task_id>/
├── task_config.json            # Placed by orchestrator (task definition)
├── env_config_initial.json     # Initial VM connection
├── env_config_golden.json      # Golden VM connection
├── initial_setup.py            # Written by setup-gen
├── golden_patch.py             # Written by setup-gen
├── reward.py                   # Written by reward-gen (copied back from sandbox)
├── REVIEW.md                   # Written by reward-gen (copied back from sandbox)
│
│   NOTE: Canonical task artifact files exist ONLY on each VM
│   at /home/user/. They are NEVER downloaded locally to prevent
│   reward-gen from peeking at setup-gen's outputs.

output/reward_sandbox/<task_id>/     # Isolated dir for reward-gen
├── task_config.json            # Copied from adversarial dir
├── env_config_initial.json     # Copied from adversarial dir
├── env_config_golden.json      # Copied from adversarial dir
├── REVIEW.md                   # Copied from previous round (if exists)
├── reward.py                   # Written by reward-gen
└── REVIEW.md                   # Written by reward-gen
```

**SECURITY**: Reward-gen ONLY gets the sandbox directory. It cannot see
setup-gen's scripts or data files. It must explore the VM to understand
the files. This prevents the discriminator from cheating by reading the
generator's source code.

## Workflow

### Step 0: Compute PROJECT_ROOT (MANDATORY — run this FIRST)

**Every session MUST start with this.** All subsequent paths are built from `$ROOT`.

```bash
ROOT="$(pwd)"
echo "PROJECT_ROOT=$ROOT"
ls "$ROOT/output/" >/dev/null 2>&1 || { echo "FATAL: not in CUA-Gym project root"; exit 1; }
```

Store `$ROOT` and use it for ALL paths below. **Never construct an output path without `$ROOT` prefix.**

### Step 1: Load Tasks

```bash
python3 -c "
import json
with open('$ROOT/<input_file>') as f:
    tasks = json.load(f)
print(f'Loaded {len(tasks)} tasks')
for i, t in enumerate(tasks):
    tid = t.get('task_id', f'task_{i}')
    instr = t.get('task_instruction', t.get('task', ''))[:70]
    print(f'  [{i}] {tid}: {instr}')
"
```

If a specific task index was given, process only that task. Otherwise process all.

### Step 2: For Each Task — Prepare Working Directory

```bash
TASK_ID="<task_id>"
WORK_DIR="$ROOT/output/adversarial/$TASK_ID"
mkdir -p "$WORK_DIR"
echo "Working directory: $WORK_DIR"
# Sanity check — path MUST contain /adversarial/
[[ "$WORK_DIR" == *"/adversarial/"* ]] || { echo "FATAL: WORK_DIR missing /adversarial/"; exit 1; }
```

Extract the task entry from the input file and write `task_config.json`:

```bash
python3 -c "
import json
with open('$ROOT/<input_file>') as f:
    tasks = json.load(f)
task = tasks[<index>]
config = {
    'task_id': task.get('task_id', 'task_<index>'),
    'domain': '<DOMAIN>',
    'task_instruction': task.get('task_instruction', task.get('task', '')),
    'context': task.get('context', ''),
    'difficulty': task.get('difficulty', 'medium')
}
with open('$ROOT/output/adversarial/<task_id>/task_config.json', 'w') as f:
    json.dump(config, f, indent=2)
print(f'Task config written for {config[\"task_id\"]}')"
```

### Step 2.5: Create TWO VM Environments

Create two isolated environments per task:
1. `initial_env` for `initial_setup.py`
2. `golden_env` for `golden_patch.py`

```bash
python3 scripts/env_cli.py create \
  --output $ROOT/output/adversarial/<task_id>/env_config_initial.json \
  --task-id <task_id>_initial

python3 scripts/env_cli.py create \
  --output $ROOT/output/adversarial/<task_id>/env_config_golden.json \
  --task-id <task_id>_golden
```

Both creations must succeed before entering the adversarial loop.

**Provider selection** (set in `.env` or shell before running):

*Local Docker (recommended for local development — no cloud credentials needed):*
```bash
export ENV_BACKEND=docker
# Optional overrides:
# DOCKER_ENV_IMAGE=gui-synth-env-desktop:latest
# DOCKER_ENV_PLATFORM=linux/amd64
# DOCKER_ENV_SHM_SIZE=2g
# DOCKER_ENV_READY_TIMEOUT=90
```

*Aliyun ECS (default — requires cloud credentials):*
- `ALIYUN_ACCESS_KEY_ID`, `ALIYUN_ACCESS_KEY_SECRET`
- `ALIYUN_REGION`, `ALIYUN_IMAGE_ID`, `ALIYUN_INSTANCE_TYPE`
- `ALIYUN_VSWITCH_ID`, `ALIYUN_SECURITY_GROUP_ID`
- Optional: `ALIYUN_RESOURCE_GROUP_ID`, `ALIYUN_USE_PRIVATE_IP`

If environment creation fails, skip this task and report the error.

### Step 3: Adversarial Loop

Execute the adversarial loop for **up to 5 rounds**:

```
for round = 1 to 5:

  ┌─────────────────────────────────┐
  │  1. Spawn setup-gen (Agent)     │
  │     → generates/fixes scripts   │
  │     → executes them             │
  └────────────┬────────────────────┘
               │
  ┌────────────▼────────────────────┐
  │  2. Verify setup-gen outputs    │
  │     → files exist and non-empty │
  └────────────┬────────────────────┘
               │
  ┌────────────▼────────────────────┐
  │  3. Spawn reward-gen (Agent)    │
  │     → generates reward.py       │
  │     → tests golden & initial    │
  │     → writes REVIEW.md          │
  └────────────┬────────────────────┘
               │
  ┌────────────▼────────────────────┐
  │  4. Read REVIEW.md              │
  │     → check 5 agreement conds  │
  └────────────┬────────────────────┘
               │
         PASS? ─┬── YES → AGREEMENT → break
                └── NO  → next round
```

#### 3a. Spawn Setup-Gen

Use the **Agent tool** to spawn a setup-gen subagent. Each spawn gets a fresh context for reasoning isolation.

**Round 1:**
```
Task(
  subagent_type: "general-purpose",
  description: "Setup-gen round 1 for <task_id>",
  prompt: "You are the setup-gen agent (Generator) for CUA-Gym.

FIRST: Read your full instructions from .claude/agents/setup-gen.md

THEN execute with these parameters:
  Working directory: $ROOT/output/adversarial/<task_id>/
  Round: 1
  Initial VM config: $ROOT/output/adversarial/<task_id>/env_config_initial.json
  Golden VM config: $ROOT/output/adversarial/<task_id>/env_config_golden.json

IMPORTANT: Use the EXACT absolute paths above. Do NOT shorten or modify them.
Follow all instructions in setup-gen.md. Generate initial_setup.py and golden_patch.py, execute them, and verify outputs exist."
)
```

**Round N > 1:**
```
Task(
  subagent_type: "general-purpose",
  description: "Setup-gen round <N> for <task_id>",
  prompt: "You are the setup-gen agent (Generator) for CUA-Gym.

FIRST: Read your full instructions from .claude/agents/setup-gen.md

THEN execute with these parameters:
  Working directory: $ROOT/output/adversarial/<task_id>/
  Round: <N>
  Initial VM config: $ROOT/output/adversarial/<task_id>/env_config_initial.json
  Golden VM config: $ROOT/output/adversarial/<task_id>/env_config_golden.json

IMPORTANT: Use the EXACT absolute paths above. Do NOT shorten or modify them.
Previous round FAILED verification. Read REVIEW.md in the working directory for specific feedback from the reward-gen agent. Fix ONLY the identified issues — do not rewrite from scratch unless the feedback says the approach is fundamentally wrong.

Follow all instructions in setup-gen.md."
)
```

#### 3b. Verify Setup-Gen Outputs

After the setup-gen agent returns, verify scripts exist locally:

```bash
python3 -c "
import os
workdir = '$ROOT/output/adversarial/<task_id>'
scripts = ['initial_setup.py', 'golden_patch.py']
missing = [f for f in scripts if not os.path.exists(os.path.join(workdir, f))]
if missing:
    print(f'MISSING scripts: {missing}')
else:
    for s in scripts:
        sz = os.path.getsize(os.path.join(workdir, s))
        print(f'  {s}: {sz} bytes')
    print('All setup-gen scripts present locally')
"
```

**DO NOT re-execute** `initial_setup.py` or `golden_patch.py` here. Setup-gen already executed both scripts on their respective VMs (Step 6 of setup-gen.md). Re-executing non-idempotent scripts (e.g., Slack message posting, file append) would corrupt VM state.

If scripts are missing locally, do NOT proceed to reward-gen. Re-invoke setup-gen.

#### 3c. Spawn Reward-Gen (in isolated sandbox)

**CRITICAL**: Reward-gen must NOT see setup-gen's scripts or data files. Create an isolated sandbox directory before spawning:

```bash
TASK_ID="<task_id>"
SANDBOX="$ROOT/output/reward_sandbox/$TASK_ID"
rm -rf "$SANDBOX" && mkdir -p "$SANDBOX"

# Copy ONLY task definition and VM connection — nothing else
cp "$ROOT/output/adversarial/$TASK_ID/task_config.json" "$SANDBOX/"
cp "$ROOT/output/adversarial/$TASK_ID/env_config_initial.json" "$SANDBOX/"
cp "$ROOT/output/adversarial/$TASK_ID/env_config_golden.json" "$SANDBOX/"

# Copy REVIEW.md from previous round if it exists (feedback loop)
cp "$ROOT/output/adversarial/$TASK_ID/REVIEW.md" "$SANDBOX/" 2>/dev/null

echo "Sandbox created at $SANDBOX with: $(ls $SANDBOX)"
```

**Deploy LLM Judge helper to BOTH VMs** (required for `mock_websites` domain, optional for others):

```bash
# Upload the locked-down LLM judge helper so reward.py can import it
python3 scripts/env_cli.py -c "$ROOT/output/adversarial/$TASK_ID/env_config_initial.json" upload utils/reward_judge.py /tmp/reward_judge.py
python3 scripts/env_cli.py -c "$ROOT/output/adversarial/$TASK_ID/env_config_golden.json"  upload utils/reward_judge.py /tmp/reward_judge.py

# Inject API credentials to /tmp/reward_env on the VM
python3 scripts/env_cli.py -c "$ROOT/output/adversarial/$TASK_ID/env_config_initial.json" execute \
  "printf 'OPENAI_API_KEY=$OPENAI_API_KEY\nOPENAI_BASE_URL=$OPENAI_BASE_URL\n' > /tmp/reward_env"
python3 scripts/env_cli.py -c "$ROOT/output/adversarial/$TASK_ID/env_config_golden.json" execute \
  "printf 'OPENAI_API_KEY=$OPENAI_API_KEY\nOPENAI_BASE_URL=$OPENAI_BASE_URL\n' > /tmp/reward_env"

echo "LLM judge helper and API credentials deployed to both VMs"
```

**NOTE**: `OPENAI_API_KEY` and `OPENAI_BASE_URL` must be set in your local shell (from `.env`). The `$OPENAI_API_KEY` in the `printf` command is expanded locally before being sent to the VM. On the VM, `reward_judge.py` reads `/tmp/reward_env` directly (no dependency on shell env vars). The model, temperature, and system prompt are locked inside `utils/reward_judge.py`.

Then spawn reward-gen pointing to the **sandbox** directory:

```
Task(
  subagent_type: "general-purpose",
  description: "Reward-gen round <N> for <task_id>",
  prompt: "You are the reward-gen agent (Discriminator) for CUA-Gym.

FIRST: Read your full instructions from .claude/agents/reward-gen.md

THEN execute with these parameters:
  Working directory: $ROOT/output/reward_sandbox/<task_id>/
  Round: <N>

IMPORTANT: Use the EXACT absolute paths above. Do NOT shorten or modify them.
Follow all instructions in reward-gen.md. Generate reward.py, test against golden_env and initial_env artifacts on the VM, and write REVIEW.md with your structured verdict.

Your working directory contains ONLY task_config.json, env_config_initial.json, and env_config_golden.json. You must use the VM (via env_cli.py) to explore files. Do NOT attempt to read files from output/adversarial/ or any other directory."
)
```

After reward-gen returns, copy its outputs back to the main workdir:

```bash
cp "$ROOT/output/reward_sandbox/<task_id>/reward.py" "$ROOT/output/adversarial/<task_id>/" 2>/dev/null
cp "$ROOT/output/reward_sandbox/<task_id>/REVIEW.md" "$ROOT/output/adversarial/<task_id>/" 2>/dev/null
```

#### 3d. Check Agreement

After the reward-gen agent returns (and outputs are copied back), read REVIEW.md and check the verdict:

```bash
python3 -c "
review = open('$ROOT/output/adversarial/<task_id>/REVIEW.md').read()

if '## Verdict: PASS' in review:
    print('AGREEMENT: YES')
else:
    print('AGREEMENT: NO')
    for line in review.split('\n'):
        if '❌' in line:
            print(f'  {line.strip()}')
"
```

**The 5 Agreement Conditions** (checked by reward-gen, reported in REVIEW.md):

| # | Condition | Meaning |
|---|-----------|---------|
| 1 | initial_setup.py executes on initial_env | Script runs cleanly in initial VM |
| 2 | golden_patch.py executes on golden_env | Script runs cleanly in golden VM |
| 3 | reward(golden_env) == 1.0 | Reward script confirms golden env target state |
| 4 | reward(initial_env) == 0.0 | Reward script gives zero in initial env |
| 5 | No forbidden patterns in reward.py | Reward script uses real verification |

**IMPORTANT**: Trust the structured verdict in REVIEW.md. Do NOT re-run scripts yourself or second-guess the verifier. Your job is orchestration, not verification.

### Step 4: On Agreement — Collect Final Outputs

**⚠️ ALL substeps below are MANDATORY. The task is NOT complete until config.json exists in output/final/. Do NOT stop early.**

#### 4a. Generate config.json FIRST (most critical output)

```bash
python3 -c "
import json, os
from datetime import datetime

workdir = '$ROOT/output/adversarial/<task_id>'
task = json.load(open(f'{workdir}/task_config.json'))

OSS_PREFIX = 'oss://ofasys-wlcb-toshanghai/osworld/rl/tasks/rlvr/'
TASK_PATH = task['task_id']
setup_script_url = f'{OSS_PREFIX}{TASK_PATH}/initial_setup.py'
reward_script_url = f'{OSS_PREFIX}{TASK_PATH}/reward.py'
setup_local_path = '/home/user/initial_setup.py'

final_config = {
    'instruction': task.get('task_instruction', ''),
    'id': task['task_id'],
    'app_type': task['domain'],
    'config': [
        {
            'type': 'download',
            'parameters': {
                'files': [
                    {
                        'url': setup_script_url,
                        'path': setup_local_path
                    }
                ]
            }
        },
        {
            'type': 'execute',
            'parameters': {
                'command': f'python3 {setup_local_path}'
            }
        }
    ],
    'evaluator': {
        'type': 'python',
        'url': reward_script_url
    },
    'generated_at': datetime.now().isoformat(),
    'adversarial_rounds': '<N>'
}

final_dir = f'$ROOT/output/final/{task[\"task_id\"]}'
os.makedirs(final_dir, exist_ok=True)
with open(f'{final_dir}/config.json', 'w') as f:
    json.dump(final_config, f, indent=2)
print(f'config.json written to {final_dir}/')
"
```

**IMPORTANT final-config rules**:
- Use `config` (NOT `initial_config`).
- Use `instruction`/`id` (NOT `task_instruction`/`task_id`) in the exported JSON.
- `config` must bootstrap setup via `download` + `execute` (run setup script with `python`, not `bash`).
- `evaluator` stays as `type: python` + reward script `url`.

#### 4b. Copy scripts and review

```bash
TASK_ID="<task_id>"

# Copy local files (scripts, reward, review)
cp "$ROOT/output/adversarial/$TASK_ID/initial_setup.py" "$ROOT/output/final/$TASK_ID/"
cp "$ROOT/output/adversarial/$TASK_ID/golden_patch.py" "$ROOT/output/final/$TASK_ID/"
cp "$ROOT/output/adversarial/$TASK_ID/reward.py" "$ROOT/output/final/$TASK_ID/"
cp "$ROOT/output/adversarial/$TASK_ID/REVIEW.md" "$ROOT/output/final/$TASK_ID/"
```

#### 4c. Download data files from VMs

```bash
python3 scripts/env_cli.py -c "$ROOT/output/adversarial/<task_id>/env_config_initial.json" execute "ls /home/user/<task_id>.*"
python3 scripts/env_cli.py -c "$ROOT/output/adversarial/<task_id>/env_config_golden.json"  execute "ls /home/user/<task_id>.*"

# Download canonical artifact from each env (choose ext from ls output)
python3 scripts/env_cli.py -c "$ROOT/output/adversarial/<task_id>/env_config_initial.json" download "/home/user/<task_id>.<ext>" "$ROOT/output/final/<task_id>/<task_id>_from_initial_env.<ext>"
python3 scripts/env_cli.py -c "$ROOT/output/adversarial/<task_id>/env_config_golden.json"  download "/home/user/<task_id>.<ext>" "$ROOT/output/final/<task_id>/<task_id>_from_golden_env.<ext>"
```

#### 4d. Verify final outputs (MANDATORY — do NOT skip)

```bash
python3 -c "
import os, sys
task_id = '<task_id>'
final_dir = '$ROOT/output/final/' + task_id
required = ['config.json', 'initial_setup.py', 'golden_patch.py', 'reward.py', 'REVIEW.md']

"
```

**If any file is missing, go back and fix the issue before proceeding.**

### Step 4.5: Close VMs

After the adversarial loop exits (whether PASS or FAIL), delete BOTH VMs:

```bash
python3 -c "
import sys; sys.path.insert(0, '.')
from utils.env import Env

workdir = '$ROOT/output/adversarial/<task_id>'
Env.delete_instance(f'{workdir}/env_config_initial.json')

```

For Docker: removes the container. For Aliyun: force-deletes the ECS instance.

**IMPORTANT**: Always run this step, even if the task failed. Leaked environments incur cost or consume resources.

If 5 rounds are exhausted without agreement:

```
ORCHESTRATOR FAILED: <task_id>

  Failing conditions:
```

## Error Handling

- If an Agent spawn fails or returns an error: Log the error and count it as a failed round. The next round's prompt will include context about the failure.
- If setup-gen produces no output files: Re-invoke setup-gen with explicit note about missing files.
- If reward-gen produces no REVIEW.md: Treat as a failed round with "no verdict produced".
- If task_config.json is invalid: Skip the task and report in pipeline status.
